The 7 Principles Behind AppClaw

# PRINCIPLE WHAT IT DOES EXTRA LLM? IMPACT
7 Negative Example Cache Track failures per screen — never repeat same mistake No 3 steps saved
6 Progressive Rule Disclosure Load rules just-in-time, not all at once No Faster
4 Adaptive Context Budget Scale prompt size to screen complexity No -1,200 tok/step
1 Episodic Memory Learn from past runs via trajectory store No 30-50% fewer steps
3 Few-Shot Trajectories Inject one golden example — show, don't tell No +20-40% success
2 Rolling Summarization Compress old steps, keep recent in full detail No 50 vs 2k tokens
5 Screen Fingerprinting "This is a login screen" — anchor all reasoning No ~20 extra tokens
