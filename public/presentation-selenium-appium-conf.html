Intent-Based, Not Command-Based

Tools are designed around testing intents like "find and interact with the login form" — not raw WebDriver commands like findElement + click. The AI composes multiple low-level actions from a single high-level intent.

Vision-Based Interaction

No accessibility IDs? No problem. mcp-appium supports vision mode — it captures screenshots and uses the AI's visual understanding to identify and interact with elements, just like a human tester would.

Dual Strategy: DOM + Vision

Use structured page source when available, fall back to visual recognition when not. Best of both worlds — precision when possible, resilience always.

5 Patterns That Actually Work

How AppClaw engineers context at every step of the testing loop.

🗜

Compress

50KB page source → 3KB minimal DOM. Only what matters hits the context window.

🔍

Diff

Don't resend the whole screen. Send only what changed since the last action.

🧠

Remember

Episodic memory from past runs. "Last time on this screen, swipe worked better than scroll."

🛠

Recover

Negative example cache — track failures per screen, never repeat the same mistake.

✅

Verify

After every action, re-read the screen and confirm the expected state before moving on.
